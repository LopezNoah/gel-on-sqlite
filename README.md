# sqlite-ts: Spec-Driven TypeScript + SQLite Rebuild

This is a TypeScript reimplementation shaped by the collocated upstream specs in `spec/*.md`, adapted for SQLite.

Because SQLite is embedded, this version drops the database server/backend split and runs the full parse -> compile -> SQL -> execute path in one process.

## What is Included

- Immutable schema snapshots and delta application (`src/schema/schema.ts`)
- Declarative Gel-like schema templates (`src/schema/declarative.ts`)
- SQLite schema migration planning and SQL rendering (`src/schema/migrations.ts`)
- Reference Gel-style schema template (`src/schema/gel_schema.ts`)
- EdgeQL-like tokenizer + parser for `select`, `insert`, `update`, and `delete` (`src/edgeql/*`)
- Semantic compilation from AST to IR (`src/compiler/semantic.ts`)
- IR-to-SQL compilation for SQLite (`src/sql/compiler.ts`)
- Runtime execution engine (`src/runtime/engine.ts`)
- Optional HTTP interface for query execution (`src/http/server.ts`)
- Browser query playground with editable examples, query suggestions, data browser link traversal, and AST/IR/SQL/result step inspection (`GET /`)
- Schema page with Studio-like type explorer, migration plan/SQL preview, schema graph, and local draft history (`GET /types`)
- Requirement traceability matrix (`spec/traceability.ts`)
- Spec task checklist (`spec/TASKS.md`)
- sqlite-ts spec workspace (`spec/README.md`)

## Spec Mapping (Adapted)

- `SPEC-001` query lifecycle -> `src/edgeql/parser.ts`, `src/compiler/semantic.ts`, `src/sql/compiler.ts`, `src/runtime/engine.ts`
- `SPEC-010` immutable schema model -> `src/schema/schema.ts`
- `SPEC-020` AST/parser with error locations -> `src/edgeql/tokenizer.ts`, `src/edgeql/parser.ts`
- `SPEC-021` semantic compilation -> `src/compiler/semantic.ts`
- `SPEC-030/031/032` backend lowering -> `src/ir/model.ts`, `src/sql/compiler.ts`
- `SPEC-040/043` runtime and HTTP API -> `src/runtime/*`, `src/http/server.ts`
- `SPEC-070` traceability mapping -> `spec/traceability.ts`, `tests/traceability.test.ts`

## Declarative Schema Example

```ts
import { GEL_REFERENCE_SCHEMA } from "./src/schema/gel_schema.js";
import { renderSchemaSQL } from "./src/schema/migrations.js";

const ddl = renderSchemaSQL(GEL_REFERENCE_SCHEMA);
console.log(ddl);
```

## Run

```bash
npm install
npm --prefix ui install
npm run test
npm run ui:build
npm run dev
```

Default server: `http://localhost:4000`

Playground UI: `http://localhost:4000/`

Schema management UI: `http://localhost:4000/types`

For frontend development with Astro:

```bash
npm run dev:all
```

This starts both the API server and Astro dev server in one terminal. Press `Ctrl+C` to stop both.

If you prefer separate terminals:

```bash
npm run dev
npm run ui:dev
```

`ui:dev` proxies `/query`, `/schema`, `/schema/plan`, and `/health` to `http://127.0.0.1:4000`, so the Astro UI stays connected to the sqlite-ts API during development.

Schema HTTP endpoints:

- `GET /schema` - returns runtime types (fields + links).
- `GET /schema/source` - returns current declarative schema source.
- `POST /schema/plan` - dry-run migration planning and generated SQL preview.
- `POST /schema/apply` - applies declarative schema source and returns updated types + migration plan.

Note: object `id` values are generated by SQLite/runtime and cannot be set in `insert`/`update` queries.

Example query body:

```json
{
  "query": "insert default::User { name := 'Noah', email := 'noah@example.com' };",
  "includeSteps": true
}
```
